ابدأ من هنا — البرومبت الكامل لـ Codex
========================================

بعد فك ضغط المشروع وفتح مجلد QoyodArchive داخل Codex، انسخ كل النص التالي وأرسله مرة واحدة:

أنت مسؤول عن تشغيل واستكمال تطوير مشروع حقيقي اسمه «نظام أرشفة القيود».

ابدأ قبل أي تعديل بالترتيب:
1. اقرأ README.md كاملًا.
2. اقرأ docs/FIX_LOG.md كاملًا.
3. اقرأ docs/PORTABLE-MANIFEST.txt وdocs/PORTABLE-CHECKLIST.md.
4. افحص app.py وconfig.py ومجلد static وocr_service والاختبارات.
5. افحص قاعدة البيانات والمرفقات قراءة فقط.
6. شغّل GET /health واختبارات المشروع.

قواعد حماية البيانات:
- ممنوع حذف أو استبدال data\archive.db.
- ممنوع حذف data\attachments أو data\uploads\originals أو data\uploads\excel.
- ممنوع استبدال .env دون نسخة احتياطية.
- قبل أي تعديل بيانات خذ نسخة احتياطية من data\archive.db ومجلد data.
- لا تضع بيانات اختبار داخل قاعدة البيانات الحقيقية؛ استخدم نسخة بيانات معزولة.
- لا تنقل .venv بين الأجهزة؛ شغّل setup.bat على كل جهاز.

الخصوصية:
- الوضع الأساسي OCR_PROVIDER=local ويستخدم Tesseract محليًا.
- لا ترسل أي مستند إلى الإنترنت أو Cloud OCR دون موافقة صريحة من صاحب المشروع.
- لا تضع API keys داخل HTML أو JavaScript؛ المفاتيح الاختيارية في .env فقط.
- لا تنشئ Firewall rule للمنفذ 8787 دون موافقة صريحة.

دورة العمل الحالية التي يجب الحفاظ عليها:
- ملف «كشف الأرشيف» هو القائمة النهائية المعتمدة.
- لا يظهر إلا القيد الموجود في الكشف.
- لا توجد خانة موجود/غير موجود ورقيًا، ولا checkbox للورق، ولا اختيار «الدور الحالي».
- كل الوظائف متاحة للجميع، والاسم مطلوب للتوثيق عند الرفع والمراجعة والقرارات.
- توجد شاشة دخول بالاسم، ويجب إرسال الهوية مع كل API request وتسجيل كل بحث وفتح ورفع ومراجعة وتحميل تقرير مع IP.
- توجد لوحة للمراجع وسجل نشاط قابل للتصفية والتصدير.
- ينشئ النظام نسخة احتياطية يومية داخل data\backups\daily عند التشغيل.
- تحديث كشف الأرشيف ينشئ Backup تلقائيًا، ويجب أن يمنع حذف قيد له مرفقات.
- لا يجوز رفع مرفق أو إضافة ملاحظة لرقم قيد غير موجود في الكشف.
- لا تُعد إضافة القيد اليدوي؛ كشف الأرشيف هو المصدر الوحيد.
- الجدول يعرض: رقم القيد، النوع، التاريخ، Num، Name، Memo، اسم المحاسب، المبلغ، الحالة، وزر «افتح».
- المبلغ يظهر بأرقام إنجليزية مثل 10,000.00 EGP.
- زر «افتح» يفتح شاشة القيد حتى إن لم توجد مرفقات.
- كل مرفقات القيد تظهر تحت بعضها داخل Scroll واحد، وليس صورة واحدة مع تنقل يدوي.
- شاشة «المراجعة والمطابقة» تخص المراجع، وتفحص رقم القيد والمبلغ والتاريخ والجهة وتكرار الملف وجودة OCR.
- نتيجة الفحص مساعدة، والقرار النهائي للمراجع.
- الواجهة عربية RTL ومريحة للعين، وبها Hover وAnimations هادئة مع prefers-reduced-motion.
- تستخدم الواجهة شعاري MMG وALMOTAHEDA محليًا فقط، ولا تستخدم شعار Elsewhere.

تشغيل جهاز جديد:
1. فك الضغط في C:\QoyodArchive.
2. تأكد من Python 3.11 أو أحدث.
3. شغّل setup.bat مرة واحدة.
4. تأكد من Tesseract في C:\Program Files\Tesseract-OCR\tesseract.exe.
5. بيانات ara وeng موجودة داخل tessdata في المشروع.
6. شغّل run.bat وافتح http://127.0.0.1:8787.
7. للشبكة استخدم run-network.bat ثم http://SERVER-IP:8787.

بعد أي تعديل:
- شغّل py_compile والاختبارات وpip check.
- افحص PRAGMA integrity_check وforeign_key_check.
- اختبر دورة المستخدم على نسخة بيانات معزولة: رفع، تكرار، OCR، مطابقة، قرار مراجع، ملاحظة، وتقرير Excel.
- اختبر الواجهة من المتصفح على Desktop وMobile.
- تأكد من عدم وجود أخطاء Browser Console.
- لا تترك بيانات أو ملفات اختبار في البيانات الحقيقية.
- اشرح بالعربية ما تغيّر وما نجح وما يحتاج قرار المستخدم.
