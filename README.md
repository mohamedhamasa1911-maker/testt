# نظام أرشفة القيود

نظام عربي محلي لحفظ مرفقات القيود، البحث في كشف الأرشيف، عرض كل صور وملفات القيد متتالية، تشغيل OCR محلي، ومساعدة المراجع في مطابقة رقم القيد والمبلغ والتاريخ والجهة.

## دورة العمل الحالية

- ملف «كشف الأرشيف» هو القائمة النهائية المعتمدة.
- لا يظهر في النظام إلا رقم قيد موجود في هذا الكشف.
- لا توجد خانة «موجود/غير موجود ورقيًا» ولا اختيار للدور الحالي.
- كل الوظائف متاحة للجميع، لكن اسم منفذ الرفع أو المراجعة مطلوب للتوثيق.
- يطلب النظام اسم المستخدم عند الدخول، ويسجل كل بحث وفتح ورفع ومراجعة وتحميل تقرير مع الوقت وIP الجهاز.
- جدول القيود يعرض: رقم القيد، النوع، التاريخ، Num، Name، Memo، اسم المحاسب، المبلغ، الحالة، وزر «افتح».
- شاشة القيد تعرض كل المرفقات تحت بعضها داخل Scroll واحد.
- «المراجعة والمطابقة» تفحص رقم القيد والمبلغ والتاريخ والجهة والتكرار وجودة OCR، ثم يظل القرار النهائي للمراجع.
- «لوحة المراجع» تعرض الأولويات والمخاطر والقيود الجاهزة.
- «سجل النشاط» يعرض من قام بكل حركة وماذا فتح، ويمكن تصديره إلى Excel.

## التشغيل على جهاز جديد

1. فك الضغط في مسار بسيط مثل `C:\QoyodArchive`.
2. لا تنقل مجلد `.venv` من جهاز آخر.
3. تأكد من تثبيت Python 3.11 أو أحدث.
4. شغّل `setup.bat` مرة واحدة.
5. ثبّت Tesseract في:

```text
C:\Program Files\Tesseract-OCR\tesseract.exe
```

6. بيانات اللغات موجودة داخل مجلد المشروع `tessdata`، ومنها `ara.traineddata` و`eng.traineddata`.
7. شغّل `run.bat`.
8. افتح `http://127.0.0.1:8787`.

للتشغيل داخل شبكة الشركة شغّل `run-network.bat` على جهاز السيرفر، ثم افتح:

```text
http://SERVER-IP:8787
```

لا تنشئ قاعدة Windows Firewall للمنفذ TCP 8787 إلا بعد موافقة صاحب المشروع.

## تحديث كشف الأرشيف

من شاشة «تحديث كشف الأرشيف» ارفع Excel أو CSV النهائي. النظام:

- يجمع صفوف نفس رقم القيد حسب `Trans #`.
- يقرأ Type وDate وNum وName وMemo واسم المحاسب.
- يحسب المبلغ من Amount أو Debit/Credit.
- ينشئ نسخة احتياطية من قاعدة البيانات قبل اعتماد الكشف.
- يمنع اعتماد كشف يحذف قيودًا مرتبطة بمرفقات، لحماية المستندات.
- يمنع رفع مرفق أو كتابة ملاحظة لرقم غير موجود في الكشف المعتمد.

## OCR والخصوصية

الوضع الأساسي في `.env`:

```dotenv
OCR_PROVIDER=local
OCR_LANGUAGES=ara+eng
TESSDATA_DIR=./tessdata
```

هذا الوضع يعمل بدون Cloud APIs ولا يرسل المستندات إلى الإنترنت. أي مزود سحابي اختياري لا يستخدم إلا بعد موافقة صريحة، وتوضع مفاتيحه في `.env` فقط، وليس في HTML أو JavaScript.

## البيانات المهمة

- قاعدة البيانات: `data\archive.db`
- المرفقات القديمة: `data\attachments`
- المرفقات الجديدة: `data\uploads\originals`
- ملفات Excel المحفوظة: `data\uploads\excel`
- التقارير: `data\exports\reports`
- النسخ الاحتياطية: `data\backups`
- النسخة اليومية التلقائية: `data\backups\daily`
- السجل: `data\logs\archive.log`

لا تحذف أو تستبدل `data\archive.db` أو مجلدات المرفقات أو `.env` دون نسخة احتياطية. قبل أي تعديل بيانات، انسخ مجلد `data` بالكامل.

## الاختبار بعد النقل

- افتح `/health` وتأكد أن `status=healthy`.
- ابحث عن قيد برقم القيد.
- جرّب فلتر اسم المحاسب والحالة.
- افتح شاشة القيد وتأكد أن كل PDF والصور تظهر متتالية.
- ارفع PDF أو صورة مع كتابة اسم منفذ العملية.
- شغّل OCR المحلي والمراجعة.
- استخرج تقرير Excel.
- تأكد أن قاعدة البيانات والمرفقات القديمة سليمة.

## ملاحظات أمان

- النسخة الحالية مناسبة لشبكة داخلية موثوقة.
- الدخول الحالي هو تعريف بالاسم للتوثيق، وليس حسابًا بكلمة مرور.
- لا يوجد HTTPS حتى الآن.
- الروابط الداخلية والمحجوزة ممنوعة عند جلب مرفق من URL.
- لا تُجرى اختبارات تعديل على قاعدة البيانات الحقيقية؛ استخدم نسخة معزولة.
